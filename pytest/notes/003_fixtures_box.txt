# приспособы из коробки
Фикстуры tmp_path и tmp_path_factory используются для создания временных каталогов.
Фикстура области функции tmp_path возвращает экземпляр pathlib.Path, который указывает на временный каталог, который остается на протяжении всего теста и немного дольше.
Фикстура области сеанса tmp_path_factory возвращает объект empPathFactory. Этот объект имеет функцию mktemp(), которая возвращает объекты Path.

• tmp_path_factory — область действия сеанса.
• tmp_path — область действия функции.
=================================
Фикстура capsys позволяет захватывать записи в stdout и stderr
=================================
«Monkey_patch» — это динамическая модификация класса или модуля во время выполнения, она предоставляет следующие функции:
• setattr(target, name, value, raise=True) — устанавливает атрибут
• delattr(target, name, raise=True) — удаляет атрибут
• setitem(dic, name, value) — устанавливает запись словаря
• delitem(dic, name, raise=True) — удаляет запись словаря
• setenv(name, value, prepend=None) — устанавливает переменную среды
• delenv(name, raise=True) — удаляет переменную среды
• syspath_prepend(path) — добавляет путь к sys.path, который является списком мест импорта Python
• chdir(path) — изменяет текущий рабочий каталог
Параметр raise сообщает pytest, следует ли вызывать исключение, если элемент еще не существует. Параметр prepend для setenv() может быть символом. Если он установлен, значение переменной среды будет изменено на значение + препенд + <старое значение>.

другие встроенные приспособы:
• capfd, capfdbinary, capsysbinary — варианты capsys, которые работают с файловыми дескрипторами и/или двоичным выводом
• caplog — похож на capsys и т. п.; используется для сообщений, созданных с помощью системы журналирования Python
• cache — используется для хранения и извлечения значений между запусками pytest. Самая полезная часть этой фикстуры заключается в том, что она позволяет использовать флаги --last-failed, --failed-first и подобные.
• doctest_namespace — полезно, если вы хотите использовать pytest для запуска тестов в стиле doctest
• pytestconfig — используется для получения доступа к значениям конфигурации, pluginmanager и хукам плагинов
• record_property, record_testsuite_property — используется для добавления дополнительных свойств в тест или тестовый набор. Особенно полезно для добавления данных в XML-отчет, который будет использоваться инструментами непрерывной интеграции
• recwarn — используется для проверки предупреждающих сообщений
• request — используется для предоставления информации о выполнении тестовой функции. Чаще всего используется во время параметризации фикстуры
• pytester, testdir — используется для предоставления временного тестового каталога для помощи в запуске и тестировании плагинов pytest. pytester — это основанная на pathlib замена для testdir на основе py.path.
• tmpdir, tmpdir_factory — аналогично tmp_path и tmp_path_factory; используется для возврата объекта py.path.local вместо объекта pathlib.Path
